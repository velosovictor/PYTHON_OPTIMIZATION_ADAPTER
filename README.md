**Pyomo Optimizer User Interface** is a powerful and user-friendly Python framework for solving **Mixed-Integer Nonlinear Programming (MINLP)** problems. Built on top of **Pyomo** with **Generalized Disjunctive Programming (GDP)** support, it provides an intuitive interface for complex optimization problems with discrete logic constraints, lookup tables, and real-time visualization.

## ‚ú® Key Features

- **üéØ Advanced MINLP Solving**: Handle complex mixed-integer nonlinear programming with discrete logic
- **üß† Constraint Theory Analysis**: Automatic degrees of freedom analysis and system classification
- **‚ö° Dual Solution Modes**: Monolithic (full-horizon) and timewise (step-by-step) simulation
- **üß© Generalized Disjunctive Programming**: Native support for logical constraints and switching behavior
- **üìä Interactive Visualization**: Real-time plotting with matplotlib integration
- **üìà Smart Lookup Tables**: Piecewise linear functions with automatic discretization
- **üîß Multi-Solver Support**: SCIP, IPOPT, and other Pyomo-compatible solvers
- **üì¶ JSON Configuration**: Simple, declarative problem specification
- **üîÑ Live Parameter Updates**: Dynamic system modification during simulation
- **üé® Flat Architecture**: Clean, maintainable codebase with minimal dependencies

## üèóÔ∏è Architecture

### Core Components

1. **üßÆ Equation Handler** (`equations.py`): Loads and processes symbolic equations from JSON
2. **üî® Model Builder** (`build_global_model.py`): Constructs comprehensive Pyomo optimization models
3. **‚öôÔ∏è Discretization Engine** (`discretization.py`): Converts ODEs to algebraic constraints using backward Euler
4. **üß† Solver Interface** (`solver.py`): Multi-solver support with automatic configuration
5. **üìä Visualization Suite** (`plotting*.py`): Real-time and post-processing visualization
6. **üîß Configuration Manager** (`parameters.py`): Robust JSON-based parameter handling
7. **üß© GDP Logic Engine** (`discrete_logic.py`): Generalized Disjunctive Programming implementation
8. **üîç Constraint Analyzer** (`constraint_analyzer.py`): Degrees of freedom analysis and system classification
9. **üéØ Optimization Framework** (`optimization.py`): Multi-objective optimization with tracking capabilities

### Flat Architecture Benefits
- **üöÄ Simple Imports**: No nested package hierarchies
- **üîß Easy Maintenance**: Clear module responsibilities  
- **üì¶ Minimal Dependencies**: Focused, lightweight design
- **üéØ Direct Access**: Quick navigation and debugging

### Solution Modes

#### Monolithic Approach
- Builds the entire time horizon at once
- Optimal for problems with strong temporal coupling
- Uses global optimization strategies

#### Timewise Approach  
- Step-by-step simulation with live updates
- Enables real-time parameter modification
- Ideal for dynamic systems and live monitoring

## üîß Configuration Reference

### Problem Definition (`object_data.json`)

| Parameter | Description | Type | Example |
|-----------|-------------|------|---------|
| `unknown_parameters` | State variables to solve for | Array | `["x", "v"]` |
| `parameters` | System parameters | Object | `{"m": 500, "k": 1000}` |
| `equations` | Differential equations | Array | `["diff(x(t), t) - v(t) = 0"]` |
| `init_conditions` | Initial values | Object | `{"x0": 2.0, "v0": 0.0}` |
| `dt_value` | Time step size | Number | `0.1` |
| `final_time` | Simulation duration | Number | `10.0` |
| `solver` | Optimization solver | String | `"scip"`, `"ipopt"` |
| `solve_mode` | Solution approach | String | `"monolithic"`, `"timewise"` |
| `minlp_enabled` | Enable discrete variables | Boolean | `true` |

### Discrete Logic (Embedded in `object_data.json`)
```json
{
  "discrete_logic": {
    "logic_constraints": [
      {
        "name": "spring_stiffness_logic",
        "disjunction": [
          {
            "conditions": ["x <= 0.5"],
            "assignments": ["k_eff == k_soft"]
          },
          {
            "conditions": ["x >= 0.5"],
            "assignments": ["k_eff == k_stiff"]
          }
        ]
      }
    ]
  }
}
```

## üéì Theory & Concepts

### Constraint Theory Analysis
The framework implements automatic constraint structure analysis:

- **Fully Constrained**: `Parameters = Restrictions` ‚Üí Unique solution exists
- **Under-Constrained**: `Parameters > Restrictions` ‚Üí Multiple solutions, optimization needed
- **Over-Constrained**: `Parameters < Restrictions` ‚Üí May be infeasible

### Generalized Disjunctive Programming (GDP)
Handles logical constraints of the form:
```
IF condition THEN assignment
```
Automatically converts to mixed-integer formulations using Big-M or Hull reformulations.

### Supported Problem Classes
- **Continuous Optimization**: Pure NLP problems
- **Mixed-Integer**: MINLP with discrete decisions  
- **Logic-Constrained**: GDP with conditional behaviors
- **Multi-Objective**: Pareto optimization and weighted objectives
- **Dynamic Systems**: ODE-constrained optimization
